# 一个 “50 后”零代码基础的“老编”，如何用 AI 搭了一个世界杯数据网站

我算是一个“老媒体人”，传统媒体经历过电视，也经历过报纸，2011年，移动互联网来临顺应趋势去做了“媒体融合”，15年带团队去做了“互联网+政务服务”……读书时学的是编导和新闻，这辈子没写过一行代码。1997 年到 2000 年，我在莫干山路111号做体育记者，经历过 1998 年法国世界杯和 2000 年悉尼奥运会的后方报道——那时候写稿是看央视直播，视频接人家的开路信号，查资料用54K猫——打开一个网页需要去门口抽一支烟。

二十多年后的 这个夏天，我坐在电脑前，对着 VS Code 的黑色终端窗口，敲下了人生中第一个 `git push`。

## 起因：只是想试试 AI

事情的起点跟世界杯没什么关系。

2026 年春节休息，终于有时间琢磨当下大火的 Claude Code——一个可以对话编程的 AI 工具。软件工程出身的家里那位说，"你只要用自然语言告诉它你想做什么，它就能帮你写代码。" 虽然我也早有耳闻，但对于一个与内容生产为伍了半生的人来说，还是有点“丈二和尚”。好在近十几年经历了媒体融合、又参与了浙里办运营和城市大脑的建设，折腾新事物咱还是不怕的，我决定试试。

环境是DeepSeek帮我搭的：VS Code 怎么装、Python 怎么配置、终端是什么、`pip` 是什么意思——这些对程序员来说天经地义的东西，对我来说每一个都是一道坎。我用的是 Windows，光是"PowerShell 和 CMD 有什么区别"花了半个下午也没搞清楚，好在IDE 框架会帮你都配好。我配环境的那段时间，VSCode还没有完全集成CC switch，是家里那位援手才配好了VSCode+Claude Code+DeepSeek,CC switch是关键的关键，通过它才配置好了DeepSeek的V4-1M模型,没有这个，纯粹烧CC原生的Token早就把我烧得打退堂鼓了。

磕磕绊绊把环境跑通之后，我开始各种练手，先是和它一起闲话一些问题，比如最近几年雨带北移的话题以及未来可能给北半球带来的影响，还比如中华经典文化资源对于人工智能大模型训练的重要性，文化高质量数据集怎么规划、怎么建设、要规避哪些坑，林林总总，反正，闲了就开启个“人机对话”，漫无目的，全凭一时兴起，但话题总少不了围绕工作相关和AI相关的话题。

真正发心做一个互联网”产品”是到了6月初的事儿，眼瞅着美加墨世界杯开赛在即，但对于一个1998年经历了法国世界杯，并且参与赛事后方报道的体育记者来说，我竟然疏于关心即将开赛的足球盛事，甚至报不出48支球队有哪些。几番思索，行动从赛前的周末开始，"我想搭一个世界杯数据看板。"  我在 Claude Code 里敲下了这句话。

现在回想起来，这句话的分量我当时完全没意识到。

## 从零到一：AI 写的代码跑起来了

第一条指令很简单："帮我写一个 HTML 页面，能显示世界杯比赛数据。"

AI 瞬间生成了一段代码。我复制粘贴到浏览器里——**一片空白**。

"为什么看不到东西？"

AI 回答："可能是因为没有 CSS 样式。"

"CSS 是什么？"

这就是我在接下来的夜晚跟 AI 对话的日常。我就像一个刚学会走路的小孩，每一步都是新的——HTML 是骨架，CSS 是衣服，JavaScript 是肌肉，JSON 是血液。而这些概念，AI 用了大概三句话就给我讲明白了。说实话，这比 00 年代我在报社带实习生还高效。

第一版看板跑起来的那天——一个有标题、有表格、有绿色背景的简陋页面——我盯着屏幕看了十分钟。不是因为好看，是因为 **我真的做到了**。一个 50 多岁、文科出身、连 `print` 、Python都不知道什么意思的人，让一段代码在浏览器里跑了起来。

此后这个看板一边跑一边改，代码从几百行涨到几千行。因为带团队做浙里办期间，知道一个互联网产品需要从需求到开发、到测试再到上线验收的基本流程，从用户需求出发，我写下了最新战报、小组积分、球队资料三个标签，随着比赛开赛，我一边跟踪比赛比分一边优化产品界面，最后标签页增加到了七个——猜冠军、最新战报、淘汰赛对阵、小组积分、射手榜、球队资料——还有中英文切换、手机端适配、比赛实时轮询。纯 HTML/CSS/JS，一个文件，零框架依赖。

## 踩坑：那些 AI 也搞不定的深夜

顺利的部分讲完了。下面说说真实世界的琐碎和折磨。

**数据从哪来。** 一开始我以为这是最简单的问题——"AI，帮我从 FIFA 官网爬数据。" AI 照做了，然后 FIFA 的 API 返回了一堆乱码。原来足球世界的官方数据散落在 ESPN、TheSportsDB、FIFA 三个 API 里，每个格式都不一样。球队名也不统一——ESPN 叫 "United States"，TheSportsDB 叫 "USA"，FIFA 叫 "United States of America"。不做好队名标准化，三个数据源一合并就打架。

**点球数据消失了。** 淘汰赛阶段，澳大利亚对埃及打到点球大战。ESPN API 返回的比分是 1-1——没有点球信息。我以为 API 不提供这个数据，差点手动录入。直到我在 AI 的帮助下翻了 ESPN 返回的原始 JSON，才发现每个 competitor 对象里藏着一个 `shootoutScore` 字段，文档里压根没提。原来它不是不存在，只是藏在一千多行 JSON 的第 837 行。

**淘汰赛对阵图崩了。** 有一天早上醒来，打开看板——阿根廷对佛得角的比赛凭空消失了，澳大利亚对埃及出现了两遍。查了几个小时发现问题出在我让 AI 写的一个"自动更新比分到源文件"的函数。这个函数更新一场比赛后会改变文件的行号，导致后续比赛的更新写到了错误的位置。等我发现的时候，源文件里同一个比赛已经重复了 42 行。差一点就想回滚代码放弃。

**凌晨的 GitHub Pages 404。** 项目写完想发布到 GitHub Pages 上，构建了两天死活通不过。Jekyll 在背后默默处理我的 HTML 文件但没告诉我；`CNAME` 文件把自定义域名指向了旧仓库导致 github.io URL 全部 404；legacy builder 已经被 GitHub 废弃但文档没说清楚需要切换 GitHub Actions。AI 给的每一个诊断都差一点命中，但就是没命中。最后是自己对着 Settings 页面反复尝试了两个小时才搞定的。

这些时刻——AI 回答不上来、文档搜不到、只能靠反复试错找到答案的时刻——让我意识到一个简单但重要的事实：**AI 能让一个零基础的人跑起来，但跑稳了，还是需要人的判断。**

## 决赛夜

7 月 19 日，决赛日。西班牙对阿根廷，加时赛 106 分钟，Ferran Torres 绝杀。

我在浏览器里开着两个窗口：左边是直播画面，右边是我的看板。Torres 进球落地的那一瞬间，ESPN API 在 30 秒内推送了新比分，看板上淘汰赛对阵图自动刷新——决赛那个"西班牙 vs 阿根廷"的格子，冒出来一个金色的 👑 冠军图标。

那个瞬间我有点恍惚。1998 年，我在电视台看着直播写稿子的那个夏天，如果有人告诉我：26 年后你会自己写程序，坐在家里搭一个世界杯数据网站，而帮你写代码的是一个你对着它说话就能理解你的 AI——我会觉得这是科幻小说。

但这不是科幻。它发生了。

## 给同龄人的几句心里话

这个项目最后我把全部数据（105 场比赛、309 个进球、181 名射手、完整淘汰赛对阵、60 队赔率历史）用 CC0 协议放到了 GitHub 上开源。仓库在：https://github.com/dobguski/worldcup-2026-dashboard

如果你也是 “50 后”、文科生、或者一个“老媒体人”，从没碰过代码——我想跟你说几句：

这1个多月走下来，我不觉得是自己"学会了编程"。我也不需要学会。而是 **"指挥 AI 帮我完成了一个产品"**。从需求分析、架构设计、代码实现、调试部署、到最终上线——全程 Claude Code + DeepSeek 承担了绝大多数技术工作，我负责了绝大多数决策工作。它的代码可以不假思索地写出来，但"我要做一个什么""出了问题应该先查哪里""这个方案行不行"——这些判断，AI 目前还做不了。

而这个技能——跟 AI 对话、描述清楚你的需求、判断它的回答是否靠谱、指挥它迭代改进——**不需要任何编程基础**。它需要的是：愿意尝试、不怕犯错、能清晰表达、和一点固执。

我这个“50 后” 文科男做到了。你也行。

2026 年 7 月，西班牙夺冠的今晨，我看了一眼运行了 36 天的看板，在 Claude Code 里敲了最后一行对话：

"CC，这个世界杯应用可以封板了。"



（张德君 一个曾经的媒体人）
